## How to create a new route

Routes in TanStack Start are file-based. Create a file under `src/routes/` and export a `Route` using `createFileRoute`.

```tsx src/routes/posts/$id.tsx

export const Route = createFileRoute('/posts/$id')({
  component: PostPage,
});

function PostPage() {
  const { id } = Route.useParams();
  return <div>Post: {id}</div>;
}
```

For API endpoints, use a loader or server function rather than returning HTML from the component. Nested routes inherit the parent layout automatically.

```tsx src/routes/posts/index.tsx

export const Route = createFileRoute('/posts/')({
  loader: async () => {
    return await fetchPosts();
  },
  component: PostsPage,
});

function PostsPage() {
  const posts = Route.useLoaderData();
  return (
    <ul>
      {posts.map((post) => (
        <li key={post.id}>
          <Link to="/posts/$id" params={{ id: post.id }}>{post.title}</Link>
        </li>
      ))}
    </ul>
  );
}
```

## How to use server functions

Use `createRoostStart` to bind your Roost application once and export configured
helpers for server functions and route hooks.

```ts src/roost.ts

export const {
  middleware: roostMiddleware,
  fn: roostFn,
  loader: roostLoader,
  beforeLoad: roostBeforeLoad,
} = createRoostStart({
  app: () => {
    const app = new Application({});
    app.register(CloudflareServiceProvider);
    app.register(AuthServiceProvider);
    return app;
  },
});
```

Use the configured `roostFn` for server functions. Pass an `input` validator
when the function accepts user input.

```ts src/functions/posts.ts

// No input — read-only server function
export const listPosts = roostFn(async ({ resolve }) => {
  const postService = resolve(PostService);
  return postService.findAll();
});

// Typed input — mutation server function
export const createPost = roostFn(
  { input: (d: { title: string; body: string }) => d },
  async ({ resolve, input }) => {
    const postService = resolve(PostService);
    return postService.create(input);
  }
);
```

```tsx src/routes/posts/new.tsx

export const Route = createFileRoute('/posts/new')({ component: NewPostPage });

function NewPostPage() {
  const handleSubmit = async (e: React.FormEvent<HTMLFormElement>) => {
    e.preventDefault();
    const form = new FormData(e.currentTarget);
    await createPost({ title: form.get('title') as string, body: form.get('body') as string });
  };

  return (
    <form onSubmit={handleSubmit}>
      <input name="title" placeholder="Title" />
      <textarea name="body" placeholder="Body" />
      <button type="submit">Publish</button>
    </form>
  );
}
```

## How to access the Roost container from routes

TanStack Start route loaders and `beforeLoad` hooks can run in the browser
during client navigation. Use `roostLoader` or `roostBeforeLoad` when route
logic needs Roost services so the service resolution still runs on the server.

```tsx src/routes/posts/index.tsx

export const Route = createFileRoute('/posts/')({
  loader: roostLoader(async ({ resolve }) => {
    return resolve(PostService).findAll();
  }),
  component: PostsPage,
});
```

Use `roostBeforeLoad` for guards and for projecting small, serializable route
context such as `userId` or `orgId`.

```tsx src/routes/_app.tsx

export const Route = createFileRoute('/_app')({
  beforeLoad: roostBeforeLoad(async ({ resolve, location }) => {
    const auth = resolve(AuthService);
    const session = await auth.currentSession();

    if (!session) {
      throw redirect({
        to: '/login',
        search: { redirect: location?.href ?? '/' },
      });
    }

    return {
      userId: session.user.id,
      orgId: session.organization?.id ?? null,
    };
  }),
  component: () => <Outlet />,
});
```

Use normal TanStack loaders for local, public, or isomorphic data that does not
need Roost services.

If route data depends on hidden server context such as the current user,
organization, or feature flag snapshot, expose only the small public values
that matter through `roostBeforeLoad` when the router needs to see them. After
login, logout, or organization switches, call `router.invalidate()` so TanStack
reloads affected route data.

## How to configure SSR

SSR is enabled by default in TanStack Start. Configure it in `app.config.ts`. For streaming SSR, set `renderMode` to `'stream'`.

```ts app.config.ts

export default defineConfig({
  server: {
    preset: 'cloudflare-pages',
  },
  routers: {
    ssr: {
      entry: './src/entry.server.tsx',
    },
    client: {
      entry: './src/entry.client.tsx',
    },
  },
});
```

To opt a route out of SSR and render it client-only, use the `clientOnly` loader modifier from TanStack Start. See [TanStack Start SSR docs](https://tanstack.com/start/latest/docs/framework/react/ssr) for streaming and hydration options.
