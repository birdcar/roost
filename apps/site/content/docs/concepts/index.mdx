Concept pages explain _why_ Roost is built the way it is. They cover architecture,
design trade-offs, and the thinking behind each package — reading material for when you want
to deepen your understanding rather than look up a specific API.

## Architecture

These pages explain how Roost's parts fit together and the broader decisions that shaped the framework.

- [Application Architecture](/docs/concepts/architecture) — Request lifecycle, boot sequence, and the middleware pipeline
- [Service Container](/docs/concepts/service-container) — Dependency injection, service providers, and the boot protocol
- [Edge Computing](/docs/concepts/edge-computing) — Why Cloudflare Workers, the V8 isolate model, and what the binding system is
- [Laravel-Inspired Patterns](/docs/concepts/laravel-patterns) — What Roost adopted, what it changed, and the DX philosophy
- [Testing Philosophy](/docs/concepts/testing-philosophy) — Fakes over mocks, TestClient, and integration-first testing

- [@roost/ai](/docs/concepts/ai) — Class-based agents, the agentic loop, Workers AI and no API keys
- [@roost/mcp](/docs/concepts/mcp) — What MCP is, server-side tool exposure, MCP vs AI tools
- [@roost/billing](/docs/concepts/billing) — Abstract billing interface, adapter pattern, webhook verification
